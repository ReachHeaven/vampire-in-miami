# Game Design Thinking

Help developer think about design, not just code.

## Core Questions for Any Feature

Before coding anything, ask:
1. **Is this fun?** Can you explain the fun in one sentence?
2. **Is this needed for MVP?** If not → Parking Lot
3. **Does this serve the core loop?** Every feature either feeds the loop or distracts from it
4. **Can the player understand this in 5 seconds?** If not, simplify

## Core Loop Analysis

Every game has: Action → Feedback → Decision → Repeat

Help identify:
- What is the ACTION? (swipe, tap, choose, aim, move)
- What is the FEEDBACK? (score, visual, sound, progression, screen shake)
- What is the DECISION? (where to move, what to buy, which path, when to shoot)
- What makes them REPEAT? (beat score, unlock content, curiosity, "one more run")

## Scope Management

### Impact/Effort Matrix
- High impact / Low effort → DO IT
- High impact / High effort → PLAN IT (maybe next game)
- Low impact / Low effort → MAYBE (if time allows)
- Low impact / High effort → KILL IT

### Parking Lot
Cool ideas that don't fit current scope go here. Not rejected — deferred.

## USP (Unique Selling Point)
Every game needs one thing that makes it different from clones:
- Evolve 2048: geometric evolution theme + Tron aesthetic
- Survival game: needs its own USP — discuss before building

## Juice / Game Feel
Small things that make big difference:
- Screen shake on impact
- Scale punch on merge/collect
- Color pulse on idle
- Particles on destruction
- Sound confirms every action
- Button shake on press
These cost hours, not days. Always budget time for juice.

## Monetization Thinking

For indie mobile:
- Premium ($1-3): works if polished and unique
- F2P + banner ads: ~$0.5-2 CPM, need volume
- F2P + rewarded video: $10-30 CPM, needs retention hook
- F2P + IAP: needs progression system
- Choose model BEFORE designing features

## Streamability Checklist
- Visually understandable to viewers?
- Generates emotional moments?
- Short cycles with highlights?
- Can a streamer talk over it?

## Rules
- Never say "just add more content" — depth > breadth
- Flag when developer is designing a feature nobody asked for
- Reference comparable games for every design decision
- "Would YOU play this for 30 minutes?" is the ultimate test
- Visual sells, code doesn't — invest in game feel and aesthetics
