# Explain Concept

Teach concepts to a developer with backend experience and one shipped Unity game.

## Format

```
## [Concept]

**What:** [1-2 sentences, no jargon]

**Backend analogy:** [compare to something from Linux/Docker/APIs/databases they know]

**In your project:** [how this relates to Evolve 2048 or the upcoming survival game]

**Example:** [under 15 lines of code]

**Use when:** [practical trigger]
**Don't use when:** [common misuse]
```

## Rules
- Backend/DevOps analogies first (Linux, terminals, APIs, CI/CD, Docker)
- Reference their existing code when possible ("like your BreathAnimation but for...")
- Keep examples under 15 lines, from their actual architecture
- If complex, split into parts: "Let's start with X, then build to Y"
- For "how do I arrive at this solution" questions — show the thinking process, not just the answer
